    ```md
        # Understanding `ls -l` Output
        ①    ②     ③   ④     ⑤      ⑥         ⑦            ⑧
        ↓    ↓     ↓   ↓     ↓      ↓         ↓            ↓
        drwxr-xr-x  5  root  root   4096   Dec 5 10:47   charsets
        -rw-r--r--  1  root  root  240526  Apr 24  2024  hashcat.hcstat2
        lrwxrwxrwx  1  root  root     25   Apr 24  2024  modules -> ../../lib/hashcat/modules

        ---

        ## Column Reference
        | Marker | Column Name     | Description                      | Possible Values                                      |
        | :----: | --------------- | -------------------------------- | ---------------------------------------------------- |
        | **①**  | **File Type**   | First character indicates type   | `d` = directory, `-` = regular file, `l` = symlink   |
        | **②**  | **Permissions** | Access rights (9 characters)     | `r` = read, `w` = write, `x` = execute, `-` = denied |
        | **③**  | **Link Count**  | Hard links or subdirectory count | Any positive integer                                 |
        | **④**  | **Owner**       | User who owns the file           | Username (e.g., `root`)                              |
        | **⑤**  | **Group**       | Group ownership                  | Group name (e.g., `root`)                            |
        | **⑥**  | **Size**        | File size in bytes               | Numeric value                                        |
        | **⑦**  | **Timestamp**   | Last modification date/time      | Date format varies                                   |
        | **⑧**  | **Name**        | File/directory name              | Symlinks show target with `->`                       |

        ---

        ## Key Points

        > **Permission Structure (②)**
        > ```
        > rwx   r-x   r-x
        >  ↓     ↓     ↓
        > User  Group Others
        > ```

        > **File Type Indicators (①)**
        > - `d` → Directory
        > - `-` → Regular file
        > - `l` → Symbolic link
        > - `b` → Block device
        > - `c` → Character device
    ```

- Combination to allocate all permissions to all (owner, group, user) -> `777`
- Allocate all permissions to `/usr/share/hashcat/hashcat.hcstat2`
    ```bash
        chmod 777 /usr/share/hashcat/hashcat.hcstat2
    ```
- Reset back to the default permission `rw-r--r--` of `/usr/share/hashcat/hashcat.hcstat2`
    ```bash
        chmod 644 /usr/share/hashcat/hashcat.hcstat2
    ```
- _**Note:** Second Approach approach_
- Indicators for
  - User -> `u`
  - Group -> `g`
  - Others -> `o`
- Indicators for
  - Adding a permission -> `+`
  - Setting a permission -> `=`
  - Removing a permission -> `-`
- Remove the `write` permission from user for file `/usr/share/hashcat/hashcat.hcstat2`
    ```bash
        chmod u-w /usr/share/hashcat/hashcat.hcstat2 
    ```
- Give `execute` permission to both the owner and group for file `/usr/share/hashcat/hashcat.hcstat2`
    ```bash
        chmod u+x,g+x /usr/share/hashcat/hashcat.hcstat2 
    ```
- Reset back to the default permission `rw-r--r--` of `/usr/share/hashcat/hashcat.hcstat2`
    ```bash
        chmod u=rw,g=r,o=r /usr/share/hashcat/hashcat.hcstat2 
    ```
